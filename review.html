App Review Notes

App name: Clippy

Product: Content Posting API

Scopes requested: user.info.basic, video.upload (optional: video.publish)

End‑to‑end flow

  1. The desktop app opens a browser to TikTok OAuth (PKCE). Redirect URI: http://127.0.0.1:8765/callback.
  2. After the user authorizes, the app stores the access token locally on the user’s device.
  3. The user enables “TikTok mode”, chooses a segment length (1–30 minutes), then converts a source video to 9:16.
  4. The app splits the export into segments, generates subtitles locally, and for each segment either burns or soft‑muxes subtitles.
  5. With “Auto‑post segments” enabled, the app uploads each segment to the user’s TikTok inbox/drafts via the Content Posting API.

Implementation

  • Uploads use the official upload init then file upload flow. No server; tokens stored locally.
  • We only request the minimal scopes needed for the described features.
  • Users can revoke access at any time in TikTok.

Testing instructions

  1. Launch the desktop app, open TikTok section, enter Client key, click “Connect TikTok”.
  2. Select any short video, enable “TikTok mode”, choose segment length, enable “Auto‑post segments”.
  3. Click Convert; observe segments appear in the TikTok account’s inbox/drafts.
